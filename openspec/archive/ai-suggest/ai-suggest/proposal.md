# US-9: AI 智能补全

## 概述

在 AI 生成简历内容后，自动识别内容不足的字段（如 highlights 只有 1 条、技能缺少 context），结合 JD 需求和知识库语义检索，主动推荐可补充的素材，以"建议卡片"形式展示供用户采纳。

## 动机

US-6 的 AI 生成是"一次性"的——生成什么就是什么，用户无法知道哪些内容还可以补充。真实场景中，求职者的知识库里可能有更多相关素材未被检索到，或者某些段落描述不够充分。智能补全作为"第二双眼睛"，主动发现内容缺口并推荐补充素材，让简历更完整。

## 提议变更

### 新增端点

`POST /api/generate/suggest`

请求体：
```json
{
  "structured_jd": { ... },
  "section": "experience",
  "content": {
    "experience": [
      {"company": "腾讯", "role": "算法工程师", "period": "2021-至今", "highlights": ["主导模型升级"]}
    ]
  },
  "gap_report": { ... }
}
```

响应体：
```json
{
  "ok": true,
  "data": {
    "suggestions": [
      {
        "field": "experience[0].highlights",
        "type": "add_highlight",
        "suggested_text": "设计分布式训练 pipeline，日均处理 10 亿条样本",
        "source": "知识库检索: work_notes.md (相关度 0.78)",
        "reason": "该经历只有 1 条 highlight，建议补充量化成果"
      }
    ],
    "total": 1
  }
}
```

### 补全逻辑

1. **识别内容不足**：分析生成结果中每个字段的内容充分度
   - experience: highlights 少于 2 条 → 建议补充
   - projects: description 少于 20 字 或 tech_stack 为空 → 建议补充
   - skills: 某项技能 context 为空 → 建议补充
2. **定向检索**：针对不足字段，用 JD 相关技能词在知识库中检索补充素材
3. **LLM 生成建议**：基于检索结果生成具体可采纳的建议文本
4. **限制数量**：单次最多 3 条建议，避免信息过载
5. **不编造**：建议内容严格基于检索到的知识库素材

### 前端变更

- GenerateView 生成完成后，显示"智能补全建议"区域
- 每条建议以卡片形式展示：建议内容 + 来源 + 原因
- 卡片有"采纳"和"忽略"按钮
- 采纳后内容追加到对应字段（不覆盖已有内容）
- 忽略后卡片消失

## 端点

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/generate/suggest` | POST | AI 智能补全建议 |

## 约束

- 单次建议 ≤ 3 条
- 不编造知识库中不存在的内容
- 采纳后追加内容，不覆盖已有内容
- 知识库为空时返回空建议列表（不报错）
- LLM 未配置时返回空建议列表（不报错）

## 风险

- 检索质量不稳定 → 建议内容可能不够精准，需 prompt 约束
- LLM 可能编造内容 → prompt 严格要求"仅基于检索素材"
- 建议过多影响体验 → 硬限制 ≤ 3 条
