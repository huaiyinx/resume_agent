# Change: asset-cold-start

## 变更类型
新增

## 变更范围
实现 US-1 资产冷启动功能：用户拖入旧简历（PDF/Word），系统自动解析、提取结构化数据、推断方向分支、生成初始 Git 版本树。

## 动机
资产冷启动是整个产品的入口。没有它，版本树、知识库、AI 生成都无米下锅。用户手里有多份旧简历，手动整理成本高。

## 验收标准（对齐 PRD US-1）
1. 支持 PDF 和 Word（.docx）格式拖拽上传
2. 自动提取结构化字段：基本信息、教育经历、工作经历、项目经历、技能列表
3. 多份简历间智能去重（同一公司同一岗位只保留最新版）
4. 生成初始 Git 树：Master 主干 + 自动推断的方向分支
5. 解析失败时保留原始文本并标注「需人工确认」
6. 单份简历解析 ≤ 5s（10 页以内，不含 LLM 调用）

## 涉及组件
- **后端新增**：LLM Client、PDF 解析器、Word 解析器、结构化提取器、上传/解析 API 实现、版本树 API 改为读 DB
- **前端改动**：UploadZone 接后端、VersionTree 从 API 获取数据、上传进度展示
- **依赖新增**：PyMuPDF、python-docx、openai SDK

## 回滚方案
删除新增的 llm/ 模块、parsers/ 实现、还原 resumes.py 和 tree.py 为桩实现。前端还原 UploadZone 为纯 UI 组件。移除新增依赖。
